# Открытые вопросы

На момент реализации репозиторий `tgparser` не содержал технического задания, PDF-документов
или иных материалов с требованиями к парсеру (только заголовок `# tgParser` в README).
Парсер реализован на основе типового стека (Telethon + dotenv + CSV/JSON) и стандартных
ограничений безопасного использования Telegram API. Ниже зафиксированы допущения,
которые нужно подтвердить или скорректировать:

1. **Источник данных.** Предполагается, что парсер читает историю сообщений из чатов/каналов,
   перечисленных в `TARGETS` (.env), к которым у аккаунта уже есть доступ (подписка/участие).
   Не реализован поиск/обход публичных каналов без явного списка.
2. **Объём данных.** Собираются только: id сообщения, дата, чат, id и имя отправителя, текст.
   Медиафайлы, реакции, форварды и т.п. не собираются — неясно, нужны ли они.
3. **Авторизация.** Используется пользовательская сессия Telethon (`SESSION_NAME`), а не бот.
   Если планируется работа через Bot API — нужно уточнить и переработать модуль `parser`.
4. **Формат хранения.** Поддержаны CSV и JSON через `OUTPUT_FORMAT`. SQLite не реализован,
   так как не было указаний на необходимость хранения в базе данных.
5. **Лимит сообщений.** Жёстко не лимитируется через .env (по умолчанию 100 на чат в коде).
   Нужно уточнить, требуется ли вынести лимит и/или диапазон дат в конфигурацию.
6. **Периодичность запуска.** Реализован разовый запуск (`python -m src.main`).
   Не добавлен планировщик/демон — неясно, требуется ли постоянная работа.

Если что-то из вышеперечисленного не соответствует ожиданиям — уточните, и логика будет
скорректирована.
