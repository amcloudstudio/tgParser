# Открытые вопросы

На момент первоначальной реализации репозиторий `tgparser` не содержал технического задания,
PDF-документов или иных материалов с требованиями к парсеру (только заголовок `# tgParser` в
README). Парсер был реализован на основе типового стека (Telethon + dotenv + CSV/JSON) и
стандартных ограничений безопасного использования Telegram API.

Позже в репозиторий были добавлены `target_parser.cfg`, `tgproparser.pdf` и `target parser.exe`.
Это дало больше контекста, но также выявило функциональность, которую я **сознательно не стал
реализовывать** — см. раздел "Исключённая функциональность" ниже.

## Допущения по реализованной части

1. **Источник данных.** Парсер читает историю сообщений из чатов/каналов, перечисленных в
   `TARGETS` (.env), к которым у аккаунта уже есть доступ (подписка/участие).
2. **Объём данных.** Собираются только: id сообщения, дата, чат, id и имя отправителя, текст.
   Медиафайлы, реакции, форварды и т.п. не собираются — неясно, нужны ли они.
3. **Авторизация.** Используется пользовательская сессия Telethon (`SESSION_NAME`), а не бот.
4. **Формат хранения.** Поддержаны CSV и JSON через `OUTPUT_FORMAT`. SQLite не реализован.
5. **Лимит сообщений.** Жёстко не лимитируется через .env (по умолчанию 100 на чат в коде).
6. **Периодичность запуска.** Реализован разовый запуск (`python -m src.main`), без планировщика.

## Исключённая функциональность (из `target_parser.cfg` / `tgproparser.pdf`)

При разборе добавленных файлов обнаружено следующее:

- **`TARGET_PARSER_USERS_GEO_LAT_LONG`** — поиск людей в Telegram по геолокации (lat/long) с
  выгрузкой юзернеймов в CSV. Это массовый сбор персональных данных о реальных людях без их
  согласия. **Не реализовано.**
- **`PROXY_PARSING_*`** в `target_parser.cfg` описан в контексте "мультиаккинга" и скрытия
  реального IP/geo при парсинге. **Не реализовано** в этом виде — обычная поддержка прокси для
  подключения к Telegram (например, при сетевых ограничениях у пользователя) могла бы быть
  легитимной, но в данной формулировке функция ориентирована на маскировку при многоаккаунтной
  активности, что не соответствует условиям безопасного использования из исходного задания.
- **`tgproparser.pdf`** содержит ссылки на `onlinesim.io` и `sms-activate.org` — сервисы продажи
  виртуальных номеров для SMS-верификации. Обычное назначение таких сервисов в подобных
  инструментах — создание дополнительных/одноразовых Telegram-аккаунтов в обход проверки номера
  телефона (ban evasion, мультиаккинг). **Не реализовано и не будет реализовано.**
- **`TARGET_PARSER_CHATS_CHANNELS_KEYWORDS`** — поиск каналов/чатов по ключевым словам. Сам по
  себе поиск публичных каналов по ключевым словам не нарушает приватность, но не реализован в
  текущей версии, так как не было прямого запроса на эту функцию и она тесно связана с остальной
  конфигурацией файла. Может быть добавлен отдельно, если потребуется.
- **`target parser.exe`** — исполняемый файл (~11 МБ) неизвестного происхождения. Не запускался
  и не анализировался: запуск непроверенных бинарников из репозитория является риском
  безопасности.

## Итог

Текущая реализация ограничена легитимным сценарием: чтение истории сообщений из чатов/каналов,
к которым у аккаунта есть доступ, без обхода приватности, банов, лимитов Telegram или защитных
механизмов. Если нужен keyword-поиск публичных каналов (без гео и без мультиаккинга через
SMS-сервисы) — уточните, и я добавлю эту функцию отдельно.
