# tgParser

Простой и безопасный парсер сообщений Telegram: подключается к Telegram API под вашим
аккаунтом и сохраняет историю сообщений из чатов/каналов, к которым у аккаунта уже есть
доступ, в CSV или JSON. Дополнительно умеет искать публичные каналы/чаты по ключевым словам.

> См. также [QUESTIONS.md](QUESTIONS.md) — список допущений и открытых вопросов,
> так как репозиторий не содержал готового технического задания, а часть найденной
> позже документации описывала функциональность (геопоиск людей, мультиаккинг через
> SMS-сервисы), которая не была и не будет реализована по соображениям приватности
> и безопасности.

## Что делает парсер

- Подключается к Telegram через Telethon, используя пользовательскую сессию.
- Читает сообщения только из чатов/каналов, указанных в `TARGETS` (.env), к которым
  у аккаунта уже есть доступ.
- Собирает по каждому сообщению: id, дату, чат, id и имя отправителя, текст.
- Ищет публичные каналы/чаты по ключевым словам из `TARGET_KEYWORDS` (.env), используя
  обычный поиск Telegram (тот же, что в приложении) — без вступления в чаты и без
  сбора данных об их участниках.
- Сохраняет результат в `data/output/<chat>.csv`/`.json` (по чатам из `TARGETS`) и
  `data/output/channel_search.csv`/`.json` (по результатам поиска).
- Логирует ход работы и корректно обрабатывает ошибки доступа (приватные чаты,
  несуществующие username, пустые результаты).

## Чего парсер не делает (и не должен делать)

- Не вступает в чужие/закрытые чаты и не обходит ограничения доступа.
- Не используется для спама, массовых рассылок или сбора данных без legal basis.
- Не ищет людей по геолокации и не собирает персональные данные пользователей.
- Не создаёт дополнительные/одноразовые аккаунты через сервисы SMS-верификации.
- Не пытается обходить лимиты или защитные механизмы Telegram.

## Структура проекта

```
src/
  config/        # загрузка настроек из .env
  parser/        # подключение к Telegram, чтение сообщений, поиск каналов по ключевым словам
  storage/       # сохранение в CSV/JSON
  utils/         # логирование
  main.py        # точка входа
data/output/     # результаты работы (не коммитятся)
tests/           # тесты
.env.example     # шаблон конфигурации
```

## Получение API_ID и API_HASH

1. Зайдите на https://my.telegram.org/apps под своим аккаунтом.
2. Создайте приложение (любое название/платформа подойдут).
3. Скопируйте выданные `api_id` и `api_hash`.

## Настройка `.env`

Скопируйте `.env.example` в `.env` и заполните:

```
API_ID=ваш_api_id
API_HASH=ваш_api_hash
SESSION_NAME=telegram_parser
TARGETS=@channel1,@channel2
TARGET_KEYWORDS=ключевое слово 1|ключевое слово 2
OUTPUT_FORMAT=csv
OUTPUT_DIR=data/output
```

- `TARGETS` — список username или ID чатов/каналов через запятую, к которым у вашего
  аккаунта уже есть доступ.
- `TARGET_KEYWORDS` — ключевые слова для поиска публичных каналов/чатов, через `|`.
  Можно оставить пустым, если поиск не нужен.

**Никогда не коммитьте `.env` или файлы `*.session` — они содержат секреты и доступ к аккаунту.**

## Запуск

```bash
pip install -r requirements.txt
cp .env.example .env   # и заполнить значения
python -m src.main
```

При первом запуске Telethon запросит номер телефона и код подтверждения для создания
сессии (`SESSION_NAME.session`).

## Результаты

- `data/output/<имя_чата>.csv`/`.json` — сообщения по каждому чату из `TARGETS`.
- `data/output/channel_search.csv`/`.json` — найденные публичные каналы/чаты по
  `TARGET_KEYWORDS` (id, название, username, тип).

## Тесты

```bash
pip install pytest
pytest
```

## Ограничения и правила безопасного использования

- Используйте парсер только для чатов и каналов, к которым у вас есть законный доступ.
- Соблюдайте Условия использования Telegram и применимое законодательство о персональных
  данных.
- Не используйте парсер для слежки, доксинга, спама или массового сбора данных без
  согласия участников.
- Учитывайте лимиты Telegram API — не увеличивайте частоту запросов искусственно.
